# Plan: Agent Tool & Function Registration API

**Task ID:** 260323-bs7-agent-api
**Created:** 2026-03-23
**Status:** Planning
**Priority:** High

## Problem Statement

The `Agent` struct currently has NO built-in tool registry. Tools must be passed externally via `run_with_tools(task, &tool_registry)`, which is inconvenient for users who want to register tools directly on the agent instance.

## Solution Overview

Add built-in tool registration capabilities to the `Agent` struct while maintaining backward compatibility with the existing `run_with_tools()` pattern.

## Tasks

### Task 1: Add ToolRegistry field and registration methods to Agent

**Files:**
- `crates/agent/src/agent/mod.rs`

**Action:**
1. Add `tool_registry: Option<Arc<ToolRegistry>>` field to `Agent` struct
2. Add `register_tool(&mut self, tool: Arc<dyn Tool>) -> Result<(), ToolError>` method
3. Add `add_tool(&mut self, tool: Arc<dyn Tool>)` convenience method (panics on error)
4. Add `with_tool(mut self, tool: Arc<dyn Tool>) -> Self` builder-style method
5. Update `run_with_tools()` to use internal registry if available, fall back to external parameter
6. Update `run_streaming_with_tools()` similarly
7. Add `get_tool_registry(&self) -> Option<&Arc<ToolRegistry>>` accessor

**Verify:**
- `cargo check -p agent` passes
- `cargo test -p agent` passes
- Existing tests still pass (backward compatibility)
- New tests for registration methods

### Task 2: Add function registration capability

**Files:**
- `crates/agent/src/tools/mod.rs` (add new module)
- `crates/agent/src/tools/function.rs` (new file)

**Action:**
1. Create `FunctionTool` wrapper that converts async functions to Tool trait objects
2. Implement `Tool` trait for `FunctionTool` with:
   - Auto-generated JSON schema from function signature (using `schemars` or manual)
   - Description from function doc comments or provided string
   - Execute that calls the wrapped function
3. Add `register_function<F, Args, Output>(&mut self, name: &str, func: F) -> Result<(), ToolError>` method to Agent
4. Add `fn_tool()` macro for ergonomic function registration (optional, can be deferred)

**Verify:**

## Backward Compatibility

All changes maintain backward compatibility:
- Existing `run_with_tools(task, &external_registry)` still works
- Existing `Agent::new(config, llm)` still works (registry defaults to None)
- Existing tests continue to pass

## Example Usage (After Implementation)

```rust
// Register tools directly on agent
let mut agent = Agent::new(config, llm);
agent.register_tool(Arc::new(MyTool::new()))?;

// Register a simple function
agent.register_function("add", |a: i32, b: i32| async move { a + b })?;

// Run with internal tools
let result = agent.run_with_tools("calculate 2 + 3", &agent.get_tool_registry().unwrap()).await?;

// Or use builder pattern
let agent = AgentBuilder::from_file("config.toml")?
    .with_tool(Arc::new(MyTool::new()))
    .build(None)
    .await?;
```
