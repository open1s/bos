### 01-02-01: Tool trait + basic ToolExecutor

**Verify before start:** `Agent` struct and `ToolExecutor` stub from plan 01 compile.

**Do:**
1. Create `crates/agent/src/tools/mod.rs`
2. Define `ToolError` enum:
   ```rust
   #[derive(Debug, Clone)]
   pub enum ToolError {
       NotFound(String),           // Tool name not registered
       SchemaMismatch { message: String },  // JSON args don't match schema
       ExecutionFailed(String),     // Tool panicked or returned error
       Timeout,
   }
   ```
3. Update `error.rs` to include `ToolError` variants and remove the placeholder
4. Define `Tool` trait:
   ```rust
   pub trait Tool: Send + Sync {
       fn name(&self) -> &str;
       fn description(&self) -> ToolDescription;
       fn json_schema(&self) -> serde_json::Value;
       async fn execute(&self, args: serde_json::Value) -> Result<serde_json::Value, ToolError>;
   }
   
   #[derive(Debug, Clone)]
   pub struct ToolDescription {
       pub short: String,      // One-liner for the model
       pub parameters: String, // Markdown description of parameters
   }
   ```
5. Implement `Tool` for `dyn Tool` via `Box<dyn Tool>` helper:
   ```rust
   impl Tool for Box<dyn Tool> {
       fn name(&self) -> &str { (**self).name() }
       fn description(&self) -> ToolDescription { (**self).description() }
       fn json_schema(&self) -> serde_json::Value { (**self).json_schema() }
       async fn execute(&self, args: serde_json::Value) -> Result<serde_json::Value, ToolError> { (**self).execute(args).await }
   }
   ```
6. Create `crates/agent/src/tools/registry.rs` â€” `ToolRegistry`:
   ```rust
   pub struct ToolRegistry {
       tools: HashMap<String, Arc<dyn Tool>>,
   }
   ```
   Methods:
   - `new()` â†’ empty registry
   - `register(&mut self, tool: Arc<dyn Tool>)` â†’ if name exists, return `Err("duplicate")`, else insert
   - `get(&self, name: &str) -> Option<Arc<dyn Tool>>`
   - `list(&self) -> Vec<String>` â€” all tool names
   - `execute(&self, name: &str, args: serde_json::Value) -> Result<serde_json::Value, ToolError>` â€” lookup + call `tool.execute(args)`, map errors
   - `to_openai_format(&self) -> Vec<serde_json::Value>` â€” converts all tools to OpenAI function format:
     ```json
     {
       "type": "function",
       "function": {
         "name": "...",
         "description": "...",
         "parameters": { /* json_schema */ }
       }
     }
     ```

**Verify:** `cargo test -p brainos-agent --lib -- tool` â€” implement `DummyTool` struct that always returns `"executed"`, register it, test lookup by name, test duplicate registration error.

   ```rust
   struct AddTool;
   impl Tool for AddTool {
       fn name(&self) -> &str { "add" }
       fn description(&self) -> ToolDescription {
           ToolDescription {
               short: "Add two numbers".into(),
               parameters: "a: number, b: number".into(),
           }
       }
       fn json_schema(&self) -> serde_json::Value {
           serde_json::json!({
               "type": "object",

           let b = args.get("b").and_then(|v| v.as_f64()).ok_or(ToolError::SchemaMismatch { message: "missing b".into() })?;
           Ok(serde_json::json!({ "result": a + b }))
       }
   }
   ```

### 01-02-03: Schema validation + error recovery

**Verify before start:** Tool schema translation works.

**Do:**
1. Create `crates/agent/src/tools/validator.rs`
2. `validate_args(schema: &serde_json::Value, args: &serde_json::Value) -> Result<(), ToolError>`:
   - Check all `required` fields are present
   - Check types match (`number` â†’ `Number`, `string` â†’ `String`, `boolean` â†’ `Bool`, `array` â†’ `Array`, `object` â†’ `Object`)
   - Recurse into nested objects
   - Return `ToolError::SchemaMismatch { message: "missing required field: X" }` on failure
3. Update `ToolRegistry::execute()` to validate before calling:
   ```rust
   pub async fn execute(&self, name: &str, args: serde_json::Value) -> Result<serde_json::Value, ToolError> {
       let tool = self.tools.get(name).ok_or(ToolError::NotFound(name.into()))?;
       let schema = tool.json_schema();
       validate_args(&schema, &args)?;  // Returns ToolError::SchemaMismatch on failure
       tool.execute(args).await
   }
   ```
4. JSON repair attempt: if `Parse` error on args (string instead of object), try `serde_json::from_str::<Value>(&args_str)` before failing

**Verify:** `cargo test -p brainos-agent --lib -- validation` â€” pass `{"a": "not a number"}` â†’ `SchemaMismatch` error with field name; pass `{"a": 1}` (missing `b`) â†’ `SchemaMismatch` with "missing required field: b".
