### 01-01-02: LlmClient trait + LlmResponse/LlmRequest types

**Verify before start:** `mod error` compiles.

**Do:**
1. Create `crates/agent/src/llm/mod.rs`
2. Define `LlmResponse` enum:
   ```rust
   pub enum LlmResponse {
       Text(String),                          // Accumulated text chunk
       ToolCall { name: String, args: serde_json::Value },  // LLM wants to call a tool
       Done,                                   // No more content, stop loop
   }
   ```
3. Define `OpenAiMessage` enum:
   ```rust
   pub enum OpenAiMessage {
       System { content: String },
       User { content: String },
       Assistant { content: String },
       ToolResult { tool_call_id: String, content: String },
   }
   ```
4. Define `LlmRequest` struct:
   ```rust
   pub struct LlmRequest {
       pub model: String,
       pub messages: Vec<OpenAiMessage>,
       pub tools: Option<Vec<serde_json::Value>>,  // OpenAI tool format
       pub temperature: f32,
       pub max_tokens: Option<u32>,
   }
   ```
5. Define `LlmClient` trait:
   ```rust
   pub trait LlmClient: Send + Sync {
       async fn complete(&self, req: LlmRequest) -> Result<LlmResponse, LlmError>;
       fn supports_tools(&self) -> bool;
       fn provider_name(&self) -> &'static str;
   }
   ```
6. Create `crates/agent/src/llm/client.rs` — `OpenAiClient` struct with:
   - Owns `reqwest::Client` (built once, connection pooling)
   - `new(base_url, api_key, default_model)` constructor
   - `build_request()` → builds POST to `/chat/completions`
   - `complete()` → POST, parse JSON response, extract first choice, return `LlmResponse`
   - `supports_tools()` → true
   - `provider_name()` → "openai"
7. Parse OpenAI response: handle both `message.content` (text) and `tool_calls` (array)
8. If `tool_calls` present, return first `ToolCall { name, args }`, extract `function.arguments` JSON string as args
9. If `finish_reason == "stop"`, return `Done`
10. If content present, return `Text(chunk)`

**Verify:** `cargo test -p brainos-agent --lib -- client` — implement a mock client test using `MockLlmClient` that returns hardcoded JSON responses.

### 01-01-03: MessageLog + Agent struct

**Verify before start:** `LlmClient` trait and `LlmResponse` compile.

**Do:**
1. Create `crates/agent/src/agent/mod.rs`
2. Define `Message` enum:
   ```rust
   pub enum Message {
       User(String),
       Assistant(String),
       ToolResult { name: String, content: String },
   }
   ```
3. Create `MessageLog` struct:
   ```rust
   pub struct MessageLog {
       messages: Vec<Message>,
   }
   ```
   Methods:
   - `new()` → empty log
   - `add_user(&mut self, content: String)` → push `User(content)`
   - `add_assistant(&mut self, content: String)` → push `Assistant(content)`
   - `add_tool_result(&mut self, name: String, content: String)` → push `ToolResult { name, content }`
   - `to_api_format(&self) -> Vec<OpenAiMessage>` → convert to OpenAI message format for API
   - `len(&self) -> usize`, `is_empty(&self) -> bool`
4. Define `AgentConfig` struct:
   ```rust
   pub struct AgentConfig {
       pub name: String,
       pub model: String,
       pub base_url: String,
       pub api_key: String,
       pub system_prompt: String,
       pub temperature: f32,
       pub max_tokens: Option<u32>,
       pub timeout_secs: u64,
   }
   ```
5. Define `Agent` struct:

   pub struct Agent {

   - `new(config: AgentConfig, llm: Arc<dyn LlmClient>) -> Self`

   - `run_with_tools(&mut self, task: &str, tools: &ToolRegistry) -> Result<AgentOutput, AgentError>` — runs loop, executes tools via registry

### 01-01-04: Agent reasoning loop with tool execution

**Verify before start:** `Agent::run_with_tools()` and `MessageLog` compile.

**Do:**
1. Implement `run_loop(&mut self, tools: Option<&ToolRegistry>) -> Result<AgentOutput, AgentError>`:
   - Loop (max 10 iterations to prevent infinite loops):
     1. Build `LlmRequest` from `message_log.to_api_format()` + tools
     2. Call `self.llm.complete(req)`
     3. Match on `LlmResponse`:
        - `Text(s)` → append to `MessageLog::add_assistant(s)`, continue loop
        - `ToolCall { name, args }` → execute via `ToolRegistry::execute(&name, args)`, append `ToolResult` to log, continue loop
        - `Done` → break loop
     4. If iteration limit reached, return `AgentOutput::Error("max iterations")`
   - Return `AgentOutput::Text(concatenated_text)`
2. Define `AgentOutput` enum:
   ```rust
   pub enum AgentOutput {
       Text(String),
       Error(String),
   }
   ```
3. Update lib.rs: export `Agent`, `AgentConfig`, `AgentOutput`, `MessageLog`, `Message`
4. Update `ToolRegistry` stub in plan 02 will be used here — for now, use `Option<&dyn ToolExecutor>`:
   ```rust
   pub trait ToolExecutor: Send + Sync {
       fn execute(&self, name: &str, args: serde_json::Value) -> impl Future<Output = Result<serde_json::Value, ToolError>> + Send;
       fn list_tools(&self) -> Vec<String>;
   }
   ```
   (Plan 02 implements the concrete `ToolRegistry`)

**Verify:** `cargo test -p brainos-agent --lib -- loop` — test loop with mock that returns Text → Done (single turn) and Text → ToolCall → Done (two turns).
