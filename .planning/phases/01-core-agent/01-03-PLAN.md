### 01-03-01: SSE decoder + streaming LLM client

**Verify before start:** `llm/client.rs` and `OpenAiClient` compile.

**Do:**
1. Create `crates/agent/src/streaming/mod.rs`
2. Define `SseEvent` enum:
   ```rust
   pub enum SseEvent {
       Data(String),  // JSON payload line
       Done,           // Stream ended
       Error(String),  // Parse error
   }
   ```
3. Implement `SseDecoder`:
   ```rust
   pub struct SseDecoder {
       buffer: String,
   }
   
   impl SseDecoder {
       pub fn new() -> Self { Self { buffer: String::new() } }
       
       /// Feed a byte chunk, return any complete SSE events.
       /// Lines starting with "data: " are collected.
       /// Empty line flushes buffer as one event.
       /// "data: [DONE]" → SseEvent::Done.
       pub fn decode_chunk(&mut self, chunk: &[u8]) -> Vec<SseEvent> {
           let text = String::from_utf8_lossy(chunk);
           let mut events = Vec::new();
           
           for line in text.lines() {
               if line.is_empty() {
                   if !self.buffer.is_empty() {
                       let data = std::mem::take(&mut self.buffer);
                       if data.trim() == "[DONE]" {
                           events.push(SseEvent::Done);
                       } else {
                           events.push(SseEvent::Data(data));
                       }
                   }
               } else if line.starts_with("data: ") {
                   self.buffer.push_str(&line[6..]);
               }
               // Ignore: event:, id:, retry:, comments
           }
           events
       }
   }
   ```
4. Define `StreamToken`:
   ```rust
   pub enum StreamToken {
       Text(String),       // Token text
       ToolCall { name: String, args: serde_json::Value },
       Done,
   }
   ```
5. Add `stream_complete()` to `LlmClient` trait:
   ```rust
   async fn stream_complete(&self, req: LlmRequest) -> Result<StreamToken, LlmError>;
   ```
6. Implement `OpenAiClient::stream_complete()`:
   - Same request as `complete()` but with `"stream": true`
   - Use `reqwest::Client::post()` → `.send().await` → `response.bytes_stream()`
   - For each `Bytes`, pass to `SseDecoder::decode_chunk()`
   - For each `SseEvent::Data(json)`, parse as JSON, extract delta content or tool_call
   - Yield tokens via `futures::Stream`
   - Return `Pin<Box<dyn Stream<Item = Result<StreamToken, LlmError>>`
7. Also implement `stream_complete()` on `MockLlmClient` for testing

**Verify:** `cargo test -p brainos-agent --lib -- sse` — test `SseDecoder`:
- `"data: {\"x\":1}\n\ndata: [DONE]\n\n"` → `Data("{\"x\":1}"), Done`
- `"data: {\"choices\":[{"delta":{"content":"hi"}}]}\n\n"` → `Data(...)`
- `""` (empty) → nothing
- Test buffer accumulation across multiple chunks

### 01-03-02: Agent streaming run

**Verify before start:** `SseDecoder` and `stream_complete()` compile.

**Do:**
1. Add `Agent::stream_run(&mut self, task: &str) -> impl Stream<Item = Result<StreamToken, AgentError>>`:
   - `message_log.add_user(task)`
   - Build `LlmRequest` with tools
   - Call `self.llm.stream_complete(req)`
   - For each `StreamToken`:
     - `Text(s)` → `message_log.add_assistant(s)` → yield `StreamToken::Text(s)`
     - `ToolCall` → execute via registry, add `ToolResult`, yield `StreamToken::ToolCall`
     - `Done` → break

3. Expose streaming via `tokio_stream::wrappers::ReceiverStream` or `#[tokio::stream]`

**Verify:** `cargo test -p brainos-agent --lib -- streaming` — mock `stream_complete()` that yields `Text("hello") → Text(" world") → Done`, verify agent yields tokens in order.

### 01-03-03: Config-driven agent loading

**Verify before start:** `Agent`, `AgentConfig`, and `ToolRegistry` compile.

**Do:**
1. Create `crates/agent/src/agent/config.rs`
2. Define config schema (deserializable from TOML/YAML):
   ```rust
   #[derive(Debug, Deserialize)]
   pub struct TomlAgentConfig {
       pub name: String,
       pub model: String,
       pub base_url: String,
       pub api_key: String,
       #[serde(default = "default_system_prompt")]
       pub system_prompt: String,
       #[serde(default = "default_temperature")]
       pub temperature: f32,
       pub max_tokens: Option<u32>,
       #[serde(default = "default_timeout")]
       pub timeout_secs: u64,
       pub tools: Option<Vec<TomlToolRef>>,
   }
   
   #[derive(Debug, Deserialize)]
   pub struct TomlToolRef {
       pub name: String,
       pub description: Option<String>,
       pub schema: Option<serde_json::Value>,
   }
   
   fn default_system_prompt() -> String { "You are a helpful assistant.".into() }
   fn default_temperature() -> f32 { 0.7 }
   fn default_timeout() -> u64 { 60 }
   ```
3. `impl From<TomlAgentConfig> for AgentConfig` — copy fields
4. `impl From<TomlToolRef> for ToolDefinition` (helper to create tools from config)
5. `AgentBuilder` struct:
   ```rust
   pub struct AgentBuilder {
       config: TomlAgentConfig,
       tools: Vec<Arc<dyn Tool>>,
   }
   
   impl AgentBuilder {
       pub fn from_toml(toml_str: &str) -> Result<Self, AgentError>;
       pub fn from_file(path: &Path) -> Result<Self, AgentError>;  // uses ConfigLoader
       pub fn with_tool(mut self, tool: Arc<dyn Tool>) -> Self;
       pub async fn build(self, session: Option<Arc<zenoh::Session>>) -> Result<Agent, AgentError>;
   }
   ```
6. `AgentBuilder::build()`:
   - Create `OpenAiClient` from config
   - Register all tools (including bus tools if session provided)
   - Build `Agent` from config
   - Return `Agent`

**Verify:** `cargo test -p brainos-agent --lib -- config`:
- Parse TOML: `name = "test"`, `model = "gpt-4o"` → `AgentConfig` matches
- `AgentBuilder::from_toml()` → `AgentBuilder` → `build()` → `Agent`
- Missing required field → `serde_json::Error`
